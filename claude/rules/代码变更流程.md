# 代码变更规则

修改任何模块的代码时，请遵循以下流程：

1. 向用户问好："你好，京城郭少！"，再告知用户你要大概要做的事情。
2. **修改代码** 在对应的模块目录中进行修改
3. **更新模块版本号** ：
4. **记录 changelog** 
  - 如果有changelog文件，直接变更此文件。
  - 如果没有changelog，新建 `docs/changelog_<module>.md` 或 `docs/changelog.md` 文件
  - 在文件顶部添加新版本段落(新的变更写在最前面)
  - 使用前缀：`feat`（新功能）、`fix`（修复）、`refactor`（重构）、`comment`（注释）、`optimize`（优化）

版本号规范：

* 版本号由va.b.c构成。a表示大版本号，如果没有不兼容的版本变更，一般不变。b表示feature、reflector、optimize变更。c表示小版本变更，如fix修复，comment注释，test测试用例等。
* 示例：```v1.2.30``` 、 ```v1.3.1```。

changelog文件格式示例：

```md
# v1.3.0

* feat 新增了xxx功能
* fix 修复了xxx问题

# v1.2.4

* fix 修复了xxx问题
```
